# Todoアプリ 仕様書

## 1. 概要

シンプルで軽量なTodoアプリ。タスクの追加・編集・削除・検索・フィルタリングを提供する。
データは Go API 経由でクラウド（PostgreSQL）に保存され、Google アカウントで認証してマルチデバイス間でリアルタイム同期する。

> アーキテクチャの詳細は [architecture.md](architecture.md)、API・DB・認証フローの詳細は [cloud-design.md](cloud-design.md) を参照。

## 4. 機能要件

### 4.1 タスク管理

| 機能 | 内容 |
|---|---|
| タスク追加 | タイトル・メモ・期限日・重要度・緊急度・タグ・ステータスを設定して作成 |
| タスク編集 | 各項目を後から変更可能（PUT で全置換） |
| タスク削除 | 個別削除（確認ダイアログあり） |

### 4.2 タスクの属性

| 属性 | 型 | 詳細 |
|---|---|---|
| タイトル | テキスト | 必須 |
| メモ | テキスト（任意） | |
| 期限日 | 日時（任意） | |
| 重要度 | 整数 1〜3（必須、デフォルト 1） | 1=低 / 2=中 / 3=高 |
| 緊急度 | 整数 1〜3（必須、デフォルト 1） | 1=低 / 2=中 / 3=高 |
| タグ | 複数参照 | 後述「4.4 タグ」参照 |
| ステータス | 参照（必須） | 後述「4.3 ステータス」参照 |
| 作成日時 | 日時 | サーバー側で自動設定 |
| 更新日時 | 日時 | サーバー側で自動更新 |

### 4.3 ステータス

ユーザー初回サインイン時にデフォルトデータが投入される。ユーザーが任意に追加・編集・削除可能。

| ステータス名（デフォルト） |
|---|
| 未着手 |
| 進行中 |
| 完了 |

### 4.4 タグ

- 自由入力で作成（テキスト）
- 1つのタスクに複数のタグを付与可能
- 名前はユーザー内で一意
- タグを削除した場合、そのタグを持つタスクからも除去される

### 4.5 一覧表示

- タスクを一覧表示する
- デフォルトの表示順: 作成日時の降順（新しい順）
- タスクカードに表示する情報: タイトル、期限日、重要度・緊急度、ステータス、タグ
- 「完了」ステータスのタスクはデフォルトで非表示（フィルタで表示切り替え可能）

### 4.6 フィルタリング

以下の条件で絞り込み可能。複数条件の組み合わせに対応。サーバー側でクエリパラメータとして処理。

| フィルタ項目 | 詳細 |
|---|---|
| タグ | 選択したタグを持つタスクのみ表示 |
| 重要度 | 選択した重要度レベル（1〜3）のタスクのみ表示 |
| 緊急度 | 選択した緊急度レベル（1〜3）のタスクのみ表示 |
| ステータス | 選択したステータスのタスクのみ表示（除外指定も可） |
| 期限日 | 期限切れフィルタ |
| 検索 | タイトルを対象にキーワード検索 |

### 4.7 ソート

以下の項目でソート可能（昇順・降順の切り替えあり）。

| ソート項目 |
|---|
| 作成日時（デフォルト）|
| 更新日時 |
| 期限日 |
| 重要度 |
| 緊急度 |
| タイトル |

### 4.8 リアルタイム同期

- ログイン後、WebSocket（`/ws`）に接続
- 他端末でデータが変更されると `*.changed` イベントが配信される
- クライアントは該当 Provider を invalidate して自動再取得

## 5. データモデル

スキーマの詳細（CREATE TABLE 文・ER 図）は [cloud-design.md](cloud-design.md#3-postgresql-スキーマ) を参照。

### テーブル概要

| テーブル | 説明 |
|---|---|
| `users` | Google 認証ユーザー（`google_sub` でユニーク） |
| `statuses` | ステータス（ユーザーごと） |
| `tags` | タグ（ユーザーごと・名前はユーザー内でユニーク） |
| `tasks` | タスク（ユーザーごと） |
| `task_tags` | タスク↔タグの多対多中間テーブル |

すべての PK は UUID。`user_id` FK により各ユーザーのデータが分離される（`ON DELETE CASCADE`）。
